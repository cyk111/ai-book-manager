                     ┌──────────────────────────┐

        ┌─────────────┬───────────┼───────────┬─────────────┐
        ▼             ▼           ▼           ▼             ▼

## 1. 设置面板

**入口**：Obsidian 设置 → AI Book Manager

| 配置项 | 类型 | 默认值 | 说明 |
|---|---|---|---|
| Book directory | 文本 | (空) | 书籍目录绝对路径 |
| Supported formats | 文本 | `.pdf,.epub,.txt` | 逗号分隔 |
| DeepSeek API key | 密码 | (空) | 本地存储 |
| DeepSeek API URL | 文本 | `https://api.deepseek.com/v1` | 兼容 OpenAI 格式 |
| Model | 文本 | `deepseek-chat` | 模型名 |
| Auto AI tagging | 开关 | 关闭 | 扫描后自动排队打标 |
| Notes folder | 文本 | `📚图书库` | vault 内相对路径 |
| Max scan pages | 滑块 (1-10) | 3 | AI 分类时提取的页数 |
| Max concurrency | 滑块 (1-5) | 1 | 并行 AI 请求数 |

**状态**：✅ 已实现

## 2. 书籍扫描

**触发方式**：
- 命令面板 → "Scan book directory"
- 侧边栏 → "Scan Now" 按钮
- 快捷键（用户可自定义）

**处理流程**：
```
配置目录 → 递归遍历 → 格式过滤 → SHA256 去重
    → 提取元数据 (书名/作者/前3页文本)
    → 创建 Markdown 笔记 (frontmatter + 基础信息)
    → 写入 vault
```

**去重策略**：
1. 计算每个文件的 SHA256（前 64KB）
2. 与已知书籍比对
3. 相同哈希 = 跳过
4. 不同哈希 = 视为新书

**笔记模板**：
```markdown
---
title: "书名"
author: "作者"
format: "pdf"
tags: []
book_id: "book_abc123"
file_path: "/Users/.../book.pdf"
file_size: 2048000
created: "2026-07-10T00:00:00.000Z"
---

# 书名

| Field | Value |
|-------|-------|
| **Author** | 作者 |
| **Format** | pdf |
| **File** | `book.pdf` |
| **Size** | 2.0 MB |
```

**状态**：✅ 已实现（基础扫描），⚠️ 增量扫描框架已有

## 3. AI 自动打标签

**触发方式**：
- 扫描后自动（需设置中开启 "Auto AI tagging"）
- 未来：侧边栏手动触发单本打标

**处理流程**：
```
BookRecord → 提取 (书名+作者+前300字文本) → 构造分类 Prompt
    → DeepSeek API (max_tokens=200, json_object)
    → 解析 JSON → 校验标签 → 写入 frontmatter tags
```

**AI 输入**：
```
Title: 书名
Author: 作者
Preview: 前300字内容片段
```

**AI 输出**：
```json
{
  "title": "书名",
  "author": "作者",
  "tags": ["科幻", "人工智能", "哲学"],
  "category": "科幻"
}
```

**标签规范**：
- 3-6 个标签，中文优先
- 混合风格：体裁 + 主题 + 写作风格
- 自动去重、去空格
- 最多保留 6 个

**队列机制**：
- 默认 1 并发，间隔 500ms
- 任务可暂停/恢复
- 状态持久化到 data.json
- 重启 Obsidian 后自动恢复

**Token 消耗**：
- 单次分类：~200-300 token
- 千本书全量打标：~0.2 CNY

**状态**：✅ 已实现

## 4. 按需内容生成

**触发方式**：打开任意书籍笔记 → 笔记底部 AI 按钮区域 → 点击对应按钮

### 4.1 生成书籍简介

| 属性 | 值 |
|---|---|
| 输入 | 书名 + 作者 + 正文前 3000 字 |
| 输出 | ~200 字中文简介（书籍内容、适读人群、核心收获） |
| 最大输出 token | 400 |
| 按钮标签 | 📝 Summary |

### 4.2 生成全书梗概

| 属性 | 值 |
|---|---|
| 输入 | 书名 + 作者 + 正文前 5000 字 |
| 输出 | 核心主旨、核心观点、受众、收获 |
| 最大输出 token | 800 |
| 按钮标签 | 📋 Outline |

### 4.3 生成章节分析

| 属性 | 值 |
|---|---|
| 输入 | 书名 + 目录结构 + 正文前 10000 字 |
| 输出 | 逐章结构、要点提炼、金句摘录 |
| 最大输出 token | 1200 |
| 按钮标签 | 📖 Chapter Analysis |

**按钮状态管理**：
```
[正常] ──点击──> [⏳ Generating...] ──成功──> [✅ Done]
                          │
                          └──失败──> [❌ Retry]
```

**内容持久化**：
- 生成结果作为 Markdown 二级标题（`## Summary` 等）追加到笔记末尾
- 再次点击替换已有内容
- 关闭笔记后下次打开仍然可见

**状态**：✅ 按钮注入已实现，⚠️ AI 生成调用标记为 "coming soon"

## 5. 知识图谱

**实现方式**：完全复用 Obsidian 原生图谱能力，零自定义渲染代码。

**组网机制**：
1. `tags: ["科幻", "人工智能"]` → Obsidian 自动识别为标签节点
2. 全局图谱中同标签书籍自动聚类
3. 点击标签节点 → 高亮该领域所有书籍
4. 图谱中可自由拖拽、缩放、筛选

**双向链接**（未来）：
- AI 生成 `[[同类书籍]]` 内部链接
- 手动补充参考、引申关系

**状态**：✅ 已实现（标签驱动，原生支持）
