# 需求文档 — AI Book Manager

## 1. 项目背景

个人本地电子书管理场景：书籍散落在磁盘目录中，格式混杂（PDF/EPUB/TXT），缺乏分类体系和知识关联。传统方案需要手工整理、手动分类、逐本写笔记——书籍越多，维护成本越高。

**核心理念**：用户只需指向一个书籍目录，AI 自动完成分类、标签、知识图谱构建。内容生成按需触发，不浪费算力。

## 2. 用户故事

| 编号 | 角色 | 需求 | 验收标准 |
|------|------|------|----------|
| US-01 | 用户 | 配置本地书籍目录后，一键扫描生成书库 | 扫描完成，Vault 中每本书对应一条 Markdown 笔记 |
| US-02 | 用户 | 书籍自动按内容分类打标签 | 笔记 frontmatter 的 tags 字段包含 3-6 个中文标签 |
| US-03 | 用户 | 在知识图谱中看到书籍之间的关联 | Obsidian 原生图谱中同标签书籍节点聚类 |
| US-04 | 用户 | 点击按钮生成书籍简介 | 简介写入笔记正文，200 字左右 |
| US-05 | 用户 | 点击按钮生成全书梗概 | 梗概流式输出到笔记正文 |
| US-06 | 用户 | 对单本书生成逐章拆解分析 | 按章节提炼要点、金句 |
| US-07 | 用户 | 支持本地离线 AI 模型 | 通过 Ollama 本地运行，书籍内容不上传云端 |
| US-08 | 用户 | 重复扫描不产生重复笔记 | 相同 SHA256 哈希的文件自动跳过 |

## 3. 功能需求

### 3.1 基础配置 (P0)

| ID | 需求 | 说明 |
|---|---|---|
| F-01 | 书籍目录路径配置 | 设置面板中输入绝对路径，支持 macOS/Windows |
| F-02 | 文件格式白名单 | 可勾选 .pdf / .epub / .txt，支持增删 |
| F-03 | AI 模型配置 | 填写 API Key、接口地址、模型名；支持 DeepSeek/Ollama/兼容 OpenAI 格式的服务 |
| F-04 | 笔记存放目录 | 指定 vault 内子目录，默认为 📚图书库 |

### 3.2 书籍扫描与入库 (P0)

| ID | 需求 | 说明 |
|---|---|---|
| F-05 | 递归目录扫描 | 遍历配置目录下的所有子目录 |
| F-06 | 文件信息提取 | 文件名、路径、大小、修改时间 |
| F-07 | 文本浅提取 | 只读取前 3 页用于 AI 分类，不整本读取 |
| F-08 | SHA256 去重 | 文件名 + 文件前 64KB 哈希双重去重 |
| F-09 | 增量扫描 | 二次扫描只处理新增/修改的文件 |

### 3.3 AI 多标签分类 (P0)

| ID | 需求 | 说明 |
|---|---|---|
| F-10 | 智能打标 | 传入书名+作者+文本片段，输出 3-6 个标签 |
| F-11 | 跨类目标签 | 一本书可同时属于"科幻""哲学""人工智能" |
| F-12 | 标签写入 frontmatter | 写入 YAML `tags:` 字段，Obsidian 原生识别 |
| F-13 | 限速队列 | 批量打标排队执行，可配并发数和间隔 |
| F-14 | 断点恢复 | 任务队列持久化到 data.json，插件重启继续 |

### 3.4 知识图谱 (P1, 复用原生)

| ID | 需求 | 说明 |
|---|---|---|
| F-15 | 标签聚类 | 同标签书籍在 Obsidian 原生图谱中自动聚拢 |
| F-16 | 双链连接 | AI 可生成同类书籍 `[[书名]]` 双向链接 |

### 3.5 按需内容生成 (P1)

| ID | 需求 | 说明 |
|---|---|---|
| F-17 | 生成书籍简介 | 用户点击按钮触发，~200 字中文简介，写入笔记 |
| F-18 | 生成全书梗概 | 分片提取文本，总结主旨/观点/受众/收获 |
| F-19 | 生成章节分析 | 解析目录，逐章提炼要点、金句 |
| F-20 | 懒加载 | 所有 AI 生成用户主动点击才执行，不自动触发 |
| F-21 | 重新生成 | 不满意可一键刷新 |
| F-22 | 内容持久化 | 生成结果写入 Markdown，下次打开直接显示 |

### 3.6 辅助功能 (P2)

| ID | 需求 | 说明 |
|---|---|---|
| F-23 | 侧边栏总览 | 书籍列表、扫描进度、操作日志 |
| F-24 | 命令面板 | Ctrl/Cmd+P 快捷触发扫描、测试 AI 连接 |
| F-25 | 深色/浅色主题 | 跟随 Obsidian 主题自动适配 |

## 4. 非功能需求

| ID | 类型 | 需求 | 指标 |
|---|---|---|---|
| NF-01 | 性能 | 千本书扫描不卡顿 | 单次扫描 < 30s（不含 AI 调用） |
| NF-02 | 性能 | 文本提取速度 | 单本 < 1s |
| NF-03 | 成本 | Token 消耗可控 | 单本书分类 < 300 token |
| NF-04 | 隐私 | 离线模式 | 搭配 Ollama 时书籍内容不上传云端 |
| NF-05 | 兼容 | Obsidian 版本 | ≥ 1.5.0 |
| NF-06 | 兼容 | 平台 | macOS / Windows（桌面端） |
| NF-07 | 安全 | API Key 存储 | 仅存本地 data.json，不上传第三方 |

## 5. 约束与假设

### 约束
- **仅桌面端**：Obsidian 移动端不支持 Node.js `fs` 模块
- **无 DRM 文件**：加密的 EPUB/PDF 无法解析
- **扫描版 PDF**：纯图片 PDF 无法提取文字，仅用文件名打标
- **Vault 外目录**：macOS 对 Downloads/Desktop/Documents 有沙箱保护，需用户授权

### 假设
- 用户书籍主要为文字型（非图片扫描版）
- 用户有基本的 Obsidian 使用经验
- AI 服务可用（DeepSeek API 或本地 Ollama）

## 6. 术语表

| 术语 | 定义 |
|---|---|
| Vault | Obsidian 的知识库根目录 |
| Frontmatter | Markdown 文件头部的 YAML 元数据 |
| BookRecord | 插件内部的一本书的数据结构 |
| Token | AI 模型的计费单位 |
| MarkdownPostProcessor | Obsidian 插件 API，在渲染 Markdown 时注入自定义 HTML |
